# 景别是怎么判出来的 · 从学习到套用的完整数据流

---

## 0. 先说一件要紧的事：标注上的景别，是「想拍什么」还是「拍成了什么」？

**原本是前者 —— 标签在骗人。**

字幕读的是 `shot_plan.json` 里的 `s["shot"]`，那是 **shotplan 阶段的设计意图**，不是从成片画面量出来的。而 camera 层会被两道闸夹住：

```
[诊断] 设计想要 zoom：中位=1.20 最大=4.10 | 上限 emax=1.45
[诊断] 28% 帧想推近却被 max_zoom 夹住
[camera.metrics] zoom_max=1.45
```

> 28% 的帧，设计要 zoom=4.10，实际只做到 1.45。这些帧字幕写着 `MEDIUM`，画面里其实是个远景。

现在 debug 版字幕加了第三行 **`actual:`** —— 从成片反量出来的真实景别，与设计不符时用**红字**标出：

```
WIDE / FOLLOW                ← 设计意图（来自 shot_plan）
zoom=1.70   5.0/6.0s
actual: WIDE (cover=0.61)    ← 实际拍成的（从成片反量）；不符时变红
```

**这一行才是判断"模板到底套没套上"的依据。**

## 1. 景别的判定规则（学习端与反量端共用同一套）

### 1.1 核心区分：「关节没检测到」≠「关节在画面外」

| 情况 | 含义 | 是不是景别信号 |
| --- | --- | --- |
| 被别的舞者挡住 | 遮挡（漏检） | ✗ 画面其实是远景 |
| 被画幅切掉 | **取景意图** | ✓ 那才真是中景/特写 |

同一个道理在 C位 打分的可见度项里也必须成立（否则中景帧的主体会被淘汰 → 模板学不到中景，实测中景只剩 5.9%）。

### 1.2 判「是否被裁」——三个信号任一成立

**不能只看 bbox 有没有贴画面底边。** bbox 底边来自**最低的可见关键点** —— 腿被画框切掉后，bbox 停在膝盖（如 y=955），够不到画面底边（1080），于是被误判成"全身都在画面内"→ wide。

1. bbox 贴下边缘
2. bbox 贴上边缘（头出画）
3. **解剖比例反推脚明显在画外**：

\[
y_{\text{ankle}} \approx y_{\text{hip}} + 1.9\,(y_{\text{hip}} - y_{\text{shoulder}}) \;>\; 1.15\,H_{\text{frame}}
\]

> 余量 1.15 要留足：1.9×躯干只是统计比例，个体差异会把"全身刚好占满画面"的人误算成脚在画外。

### 1.3 被裁后定档

以**最低的可见关节**为主，\(r\)（整个人相对画面的大小，含画面外部分）掰正 wide/medium 边界：

| 最低可见关节 | 判定 |
| --- | --- |
| 踝可见 | \(r<0.45\) → `extreme_wide`，否则 `wide` |
| 膝可见 | \(r>1.35\) → `medium`（人已占满画面），否则 `wide` |
| 髋可见 | `medium` |
| 髋都不可见 | `closeup` |

> 为什么不能只看 \(r\)：1.9×躯干有个体差异。
> 为什么不能只看可见关节：膝盖可见也可能是占画面 1.4 倍的中景。
> 两者结合才稳。全谱六档（远景/全景/刚好占满/切腿/腰以上/胸以上）验证通过。

代码：`learn_template._shot_from_exposure()`（学习端）· `annotate.achieved_shot()`（反量端）

## 3. 数据流上每一步会出什么问题（都踩过）

| 环节 | 症状 | 根因 |
| --- | --- | --- |
| `template.path` 为空 | 全片 `wide=100%` | 回退内置模板，其 `section_shot_dist` 曾是 `{}` → 无配额可分。现已给默认值 + 缺失告警 |
| C位 打分含 `visible` | 模板学不到中景 | 被画框切掉的主体被判"可见度低"而淘汰 → C位 给了背景全身可见的人 → 判成 wide |
| `_shot_from_exposure` 只看 bbox 贴边 | 中景只剩 5.9% | bbox 底边来自最低可见关键点，腿被切后停在膝盖，够不到画面底 |
| `z_max` = 画质预算 | 字幕写 MEDIUM、画面是远景 | 1920→1280 时预算只有 1.50，而 medium 需 3.79。**采样定律，参数调不出来** |
| 只看「段数」判景别比例 | 以为模板没生效 | 一个 wide 镜头内推/摇/滚会被切成多段。要看 `[shotplan] 逐帧景别占比` |

## 4. 你这次日志的读法

```
[shotplan] 逐帧景别占比: wide=0.496 medium=0.253 extreme_wide=0.251
[shotplan] 模板目标占比: wide=0.450 medium=0.300 extreme_wide=0.150 closeup=0.100
```
→ **模板贴得不错**（wide 0.50 vs 目标 0.45、medium 0.25 vs 0.30）。closeup 被 `group_min_shot=medium` 并进了 medium。

```
[诊断] 设计想要 zoom：中位=3.58 最大=10.37 | 上限 emax=10.00
[camera.metrics] subject_in_frame_rate=0.7351  zoom_max=9.5533
```
→ `max_zoom=10` 太猛：zoom 顶到 9.55，主体入画率掉到 **0.735**。源里的舞者只有 190px 高，放大 9.5 倍等于把 190px 拉到 1800px。**建议压到 4~5**（中位需求才 3.58）。
